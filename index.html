what is Native? What is non-Native?

Native

  • Functionality - is specific to Device Hardware and Device OS
  • Code - written in a special, lower-level language for the device and OS. Objective-C, Java
  • Performance - can be faster and take full advantage of device hardware

Non - Native

  • Functionality - is general to multiple devices' hardware and devices' OSes
  • Code - written in higher-level language. Javascript
  • Performance - slower touch response, greater memory usage, and slower computations

Choosing Native or Non - Native

  • Does the app require high performance?
  • Does the app require heavy usage of device hardware functionality?
  • How many devices does the app need to run on?
  • How many developer resources are available?

What is Cross-platform?

What do multiple devices have in common?

  • Hardware?
  • OS?
  • Functionality?
  • Screen size?
  • None of the above!

Let's try again...

What do multiple devices have in common?

  • A web browser?
  • HTML?
  • Javascript?
  • CSS?
  • All of the above!

How do I make a cross-platform app with HTML, Javascript, and CSS?

Frameworks!

  • Frameworks turn HTML, Javascript, and CSS into Native Code.
  • Your HTML can be a bunch of web pages.
  • Or it can be something more complicated, like a Javascript application. Backbone.js, Ember.js, Knockout.js
  • (Case study) When we built ohmage, we started with a simple series of pages. But when it got complex we made it into a Javascript application.

Cross-platform app compared to mobile websites

  • Cross Platform apps need to work while offline
  • Cross Platform apps can access device hardware functionality (depending on the Framework...) Camera, GPS
  • Cross Platform apps may be able to use native OS features. Notifications, Application Open/Close Events
  • Mobile websites lack the "App Look"

Some Cross-platform App Framework Solutions

Appcelerator Titanium, Trigger.io, Phonegap / Cordova

Appcelerator Titanium

An open-source cross platform mobile app framework with the Titanium SDK.

  • Features an API for native UI elements: menus, dialog boxes, etc.
  • Most apps require a Titanium license

Trigger.io

"A multiplatform tool for converting HTML5/JavaScript into applications for Android, iOS, Chrome, Firefox and Windows mobile devices. It is similar to PhoneGap, Sencha, and Appcelerator, but claims to be faster and better." - BOX UK

  • Features an API for native UI elements: menus, dialog boxes, etc.
  • Cloud-build platform, no need to build locally
  • Subscription-based access to cloud builds, reloads

Cordova / Phonegap

An open-source cross platform mobile app framework with a plugin system for access to native device APIs.

  • Features an API for native device features like Camera and Geolocation
  • No license required to build Phonegap apps locally

Installing Phonegap

In the command line:

  • cordova create folder-name
  • cd folder-name
  • cordova add platform ios
  • cordova build ios

then open platforms/ios/folder-name.xcodeproj in XCode and Run

Special Considerations

There are always exceptions...

iOS Oddities

  • "It doesn't run on my device"
  • Installing and setting up XCode
  • Developer Licensing, Provisioning Profiles, Certificates
  • App Store Submission and Approval
  • App Sandboxing and limited access to other features

Android Oddities

  • "It doesn't run on these 10,000 other devices"
  • Device Fragmentation
  • Intents
  • Hardware buttons Back, Home, Search, and Info

General Cross-Platform Oddities

  • Different UI Behaviors and User Expectations
  • Different implementations of software functionality. Local Notifications
